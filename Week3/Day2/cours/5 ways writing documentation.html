Accessing a Form

Forms are members of the special collection document.forms


1. Returning a form

Properties Meaning
document.forms
returns the collection of all <form> elements in a HTML document
document.forms.attributeValue
returns a specific form depending on its attribute value
document.forms[index]
returns a specific form depending on its index on the HTML document
document.forms[index].attribute
returns the value of the attribute of a specific form


2. Returning an element of a form

Properties Meaning
document.forms[index].elements.attributeValue
returns a specific element from a form depending on its attribute value
document.forms[index].elements[index]
returns a specific element from a form depending on its index


Example:
<form name="my">
  <input name="one" value="1">
  <input name="two" value="2">
</form>

<script>
  // get the form
  let form = document.forms.my; 
  // "<form name='my'>
  // <input name='one' value='1'>
  // <input name='two' value='2'>
  // </form>"

  // OR
  let form = document.forms[0]; // the first form on the page

  console.log(form.name) // "my"

  //  get the number of forms on the page
  document.forms.length

  // get the element
  let elem = form.elements.one; // <input name='one' value='1'>
  // OR
  let elem = form.elements[0]; // <input name='one' value='1'>


</script>


3. Accessing /Modifying an element of a Form

To access the value of an input you have to use the property input.value


Example:
<form name="my">
  <input name="one" value="1">
  <input name="two" value="2">
</form>

<script>
  let form = document.forms.my; 
  let firstElement = form.elements.one; 

  console.log(firstElement.value); // 1
  firstElement.value = 'hey' 
  console.log(firstElement.value); // "hey"
</script>


4. Specific Event Handler

1. onchange

The onchangeevent triggers when the element has finished changing.
For text inputs that means that the event occurs when it loses focus.

<input type="text" onchange="alert(this.value)">
<input type="button" value="Button">


2. oninput

The input event triggers every time after a value is modified by the user. 

<input type="text" id="input"> ONINPUT <span id="result"></span>

<script>
  let newInput =  document.getElementById('input')
  newInput.oninput = function() {
    result.innerHTML = input.value;
  };
</script>

5. select Form

Properties Goal
selectID.options
the collection of <option> subelements,
selectID.options[index].selected
the option selected depends on its index
selectID.value
the value of the currently selected <option>
selectID.selectedIndex
the index of the currently selected <option>


Example:

<select id="A">
  <option value="apple">Apple</option>
  <option value="pear">Pear</option>
  <option value="banana">Banana</option>
</select>

<script>
  //get the select form
  let dropdown = document.getElementById('A')

  // all three lines do the same thing
  dropdown.options[2].selected = true; //Banana selected
  dropdown.value = 'banana'; //Banana selected
  dropdown.selectedIndex = 2; //Banana selected

  // To check which option is selected
    console.log(dropdown.selectedIndex) // "2"
    console.log(dropdown.value) // "Banana"
</script>

6. input.checked property

Property Meaning
input.checked Check if the input is checked : return a boolean


Example 

<table id="table">

    <td>Your age:</td>

    <td>
      <label>
        <input type="radio" name="age" value="young" checked> less than 18
      </label>

      </label>

8. Form Events

We saw before the events: onchange and oninput.

We will see now the onsubmitevent.

The onsubmitevent is triggered :

  • when you click on an input of type=submit
  • when you press Enter on the input
    <form onsubmit="alert('submit!');">
      First: Enter in the input field <input type="text" value="text"><br>
      Second: Click "submit": <input type="submit" value="Submit">
    </form>


More Form Events

// The best is to use addEventListener
<form name="my">
  First: Enter in the input field <input type="text" value="text"><br>
  Second: Click "submit": <input type="submit" value="Submit">
</form>

    <script>
            let form = document.forms.my;

            form.addEventListener("submit", test)

            function test(){
              alert('submit ! ')
            }
    </script>


9. Using event.preventDefault()

Many browser events have a “default action”. event.preventDefault() send a signal that those actions should be canceled.
You should use this method to prevent a from from submitting but you click on a “Submit” button.
It prevents the page from refreshing.

// Did you see what happened on the example above ? After the form was submitted, the page was automatically refreshed. To prevent this from happening, use the event object method e.preventDefault()

<form name="my">
  First: Enter in the input field <input type="text" value="text"><br>
  Second: Click "submit": <input type="submit" value="Submit">
</form>

    <script>
            let form = document.forms.my;

            form.addEventListener("submit", test)

            function test(e){
              alert('submit ! ')
              e.preventDefault()
            }
    </script>
